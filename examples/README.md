# pinnedai examples

One directory per template. Each has:

- **`pr-description.md`** — a real PR description that pinnedai parses into a claim
- **`generated.test.ts`** — the test file pinnedai writes to `tests/pinned/`
- **`why.md`** — what kind of regression this guards against

To run any of these against your own repo:

```bash
# Copy the PR description into your PR body, then:
npx pinnedai generate --pr-id pr-<N> --description "$(cat examples/<template>/pr-description.md)"
```
