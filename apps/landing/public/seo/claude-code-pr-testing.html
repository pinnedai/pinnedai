CLAUDE CODE

Claude Code PR testing — make Claude's promises testable forever

Claude Code is shipping production PRs at the velocity of a senior engineer. Pinned turns each PR's behavioral claim into a permanent CI test — so Claude's promises don't just become commits, they become guardrails.

The pattern you've seen

Claude Code finishes a task and opens a PR. The description reads:

## Summary
- Added authentication requirement to the admin export endpoint
- Implemented idempotency for the Stripe webhook handler (keyed on event_id)
- No breaking changes to existing routes

## Files changed
- middleware.ts (auth check expanded to /api/admin/*)
- app/api/admin/export/route.ts (new endpoint)
- app/webhooks/stripe/route.ts (idempotency middleware added)

You merge it. Tests pass (Claude usually writes a few). Production looks fine. Two months later, a separate Claude PR refactors the middleware and inadvertently widens the auth check to non-admin routes — or worse, removes the idempotency check on the webhook. Nobody notices until a customer reports a duplicate charge.

How Pinned protects against this

When Claude's first PR lands, Pinned's GitHub Action parses the claims and commits permanent tests:

tests/pinned/pr-42-auth-required-api-admin-export.test.ts
tests/pinned/pr-42-idempotent-webhooks-stripe.test.ts

These tests run on every commit forever. The auth-required test fires an unauthenticated GET to /api/admin/export and asserts 401/403. The idempotent test POSTs the same payload twice with the same event_id and asserts byte-identical responses.

When a future Claude PR breaks one of these, CI fails — and the failure message includes a paste-ready prompt for Claude Code to repair the issue.

The repair loop is the magic

When a pin fails in CI, the test error message contains a structured prompt designed for Claude Code:

═══ PINNED FAILURE — paste this into Claude Code / Cursor ═══

Fix the failing pinned claim in this test file:
  Claim: "Auth required on /api/admin/export"
  Original PR: pr-42
  Route: /api/admin/export
  Expected: unauthenticated GET to /api/admin/export returns 401 or 403
  Actual: returned 200

Restore the auth check on /api/admin/export. Likely candidates:
  - middleware.ts or src/middleware.ts (Next.js / Hono / generic)
  - The route handler file for /api/admin/export
  - Auth provider config (Clerk / Auth.js / Supabase / Lucia)
Preserve authenticated behavior. Do not modify this pinned test file.

After fixing, re-run:  npx vitest run tests/pinned/pr-42-auth-required-api-admin-export.test.ts
═══════════════════════════════════════════════════════════════

You copy this block, paste it into Claude Code, Claude generates the fix using your existing subscription. You commit, CI passes, the pin stays in place to catch the next regression. Zero LLM cost to Pinned for the repair step — it's your existing Claude that does the work.

Designed for the AI-coder workflow

Pinned is not competing with Claude Code, Cursor, or Aider. It's the layer that turns their output into durable guarantees. Use whatever AI writes your code — Pinned makes the claims it makes testable forever.
