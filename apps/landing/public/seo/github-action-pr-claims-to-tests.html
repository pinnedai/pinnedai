CONCEPT

The GitHub Action that turns PR claims into permanent tests

A PR description says: "Adds auth to /api/admin/export. Rate-limits /api/users to 60 req/min. Makes the Stripe webhook idempotent." Without verification, those are just claims. Pinned makes them testable forever.

The category in one line

Claim-based regression testing for AI-coded PRs. Not an AI code reviewer. Not a test generator. A registry of behavioral promises tied to permanent CI tests.

How the workflow works

  1. You open a PR. The description contains one or more claims — testable behavioral promises.
  2. The Pinned GitHub Action parses each claim (regex first, LLM fallback via OIDC — no API key required).
  3. For each claim, it generates a Vitest file in tests/pinned/.
  4. It auto-commits the test to the PR branch (opt-out via PINNEDAI_AUTOCOMMIT=false in repo variables).
  5. It posts a PR comment showing the new pins, plus any unpinned risk surfaces the diff touched (new public routes, webhook handlers, env-file changes).
  6. You merge. The tests join your suite permanently.
  7. Forever after, if any commit regresses a pinned claim, CI fails with a back-reference to the original PR and a paste-ready Cursor/Claude repair prompt.

The three claim templates in v0.1

Claim shapeWhat gets generated
Rate-limits /api/X to N req/minBursts N+1 parallel requests, asserts ≥1 returns 429
Auth required on /api/XSingle GET without auth header, asserts 401 or 403
Makes /webhooks/X idempotent on event_idPOSTs the same payload twice, asserts byte-identical response

The LLM never writes test logic — only fills slots in deterministic templates. False-positive rate stays near zero.

What's NOT in scope (deliberately)

  • Not a code reviewer — CodeRabbit, Greptile, Qodo own that surface. Use them alongside Pinned.
  • Not a generic test generator — tests are scoped to specific claim templates, not freeform AI-written assertions. Determinism is the moat.
  • Not a dependency scanner — Snyk, Socket own that.
  • Not a runtime smoke tester — Chromatic, Percy, Argos own that.

The registry is the moat. The more pins your repo has, the more locked-in Pinned becomes — and the more behavioral protection your team gets.

Install — one command

npx pinnedai init

Writes .github/workflows/pinned.yml, creates tests/pinned/, configures GitHub Actions permissions. No account, no API key. The GitHub OIDC token is the identity.
