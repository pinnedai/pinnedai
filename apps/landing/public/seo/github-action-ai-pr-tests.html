GITHUB ACTION

The GitHub Action that turns AI-coded PR claims into tests

AI coding agents (Cursor, Claude Code, Aider, Devin) ship PRs at 10× human velocity. Their PR descriptions claim things — auth was added, a route is rate-limited, a webhook is idempotent — but reviewers don't have time to verify every claim. Pinned is a GitHub Action that turns each claim into a permanent CI test.

The problem

A typical AI-coded PR description reads:

## What this PR does

- Adds auth to the admin export endpoint
- Rate-limits /api/users to 60 req/min
- Makes the Stripe webhook idempotent on event_id

The reviewer merges it. Six months later, an unrelated refactor breaks the rate limiter. Nobody notices until production gets scraped at 3am. The postmortem traces back to PR #1247 — the one that said it added rate limiting but never had a test to enforce it.

How Pinned solves it

The Pinned GitHub Action runs on every PR open and:

  1. Parses claims from the description (regex first, LLM fallback for natural-language phrasing).
  2. Generates a test file per claim under tests/pinned/.
  3. Auto-commits to the PR branch (Free + Pro both, opt-out with PINNEDAI_AUTOCOMMIT=false).
  4. Maintains PINS.md — the human-readable registry of every claim your repo has ever made.

Example: a rate-limit claim becomes this test

The claim in the PR description:

Rate-limits /api/users to 60 req/min.

Becomes tests/pinned/pr-1247-rate-limit-api-users.test.ts:

// Pinned by pinnedai — claim from pr-1247.
// Original PR claim: "Rate-limits /api/users to 60 req/min"
// Pinned to fail if this is ever regressed.

import { describe, it, expect, beforeAll } from "vitest";

const PREVIEW_URL = process.env.PREVIEW_URL;
const ROUTE = "/api/users";
const RATE = 60;
const WINDOW = "minute";

describe("pinned: rate-limit on " + ROUTE, () => {
  beforeAll(() => {
    if (!PREVIEW_URL) throw new Error("PREVIEW_URL required");
  });

  it("returns 429 after exceeding " + RATE + " requests/" + WINDOW, async () => {
    const url = PREVIEW_URL + ROUTE;
    const statuses = await Promise.all(
      Array.from({ length: RATE + 1 }, () =>
        fetch(url, { method: "GET" }).then((r) => r.status)
      )
    );
    expect(statuses.includes(429)).toBe(true);
  });
});

That test runs in CI every commit, forever. The first PR to accidentally regress the limiter fails CI with a back-reference to PR #1247 and a paste-ready prompt for Cursor/Claude to fix it.

Install — one command

npx pinnedai init

This scaffolds .github/workflows/pinned.yml, creates tests/pinned/ with an empty PINS.md registry, and configures the right Actions permissions (id-token: write for keyless LLM extraction via GitHub OIDC, contents: write for auto-commit).

No API key, no account creation — the GitHub OIDC token is the identity. Free tier supports 25 active pins per repo.
